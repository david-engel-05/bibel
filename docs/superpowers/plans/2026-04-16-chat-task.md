# Chat-Auftrag Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Nutzer können einer Session einen persistenten "Auftrag" geben, den die KI über das gesamte Gespräch hinweg in den System-Prompt injiziert.

**Architecture:** Neue `task`-Spalte in `chat_sessions` (Supabase). Backend-Endpoints `GET /session/{id}` und `PATCH /session/{id}/task` + Injektion in den System-Prompt des `/ask`-Endpoints. Frontend-Button im Header öffnet Inline-Eingabe, die per PATCH speichert und den State sofort aktualisiert.

**Tech Stack:** FastAPI (Python), Next.js (TypeScript), Supabase (Postgres), Pydantic, pytest, React

```python
def _get_session(session_id: str, db: Client) -> dict:
    """Returns session row (id, summary, summary_upto_count, task) or raises 404."""
    result = (
        db.table("chat_sessions")
        .select("id, summary, summary_upto_count, task")
        .eq("id", session_id)
        .execute()
    )
    if not result.data:
        raise HTTPException(status_code=404, detail="Session nicht gefunden")
    return result.data[0]
```

Nach dem `@app.post("/session")`-Endpoint die zwei neuen Endpoints einfügen:

```python
@app.get("/session/{session_id}")
def get_session_info(session_id: str, db: Client = Depends(get_supabase)):
    session = _get_session(session_id, db)
    return {"task": session.get("task")}


@app.patch("/session/{session_id}/task")
def update_task(session_id: str, body: TaskRequest, db: Client = Depends(get_supabase)):
    _get_session(session_id, db)
    db.table("chat_sessions").update(
        {"task": body.task or None}
    ).eq("id", session_id).execute()
    return {"ok": True}
```
